# ZRQ 公网后端与硬件数据接入计划

> 状态：进行中<br>
> Web 分支：`tkx`<br>
> 目标地址：`http://47.103.152.175:8080`

## 已确认的契约

- 正式 Web API 前缀为 `/api/v1`；前端基础地址仅配置主机与端口，不能重复写入 `/api/v1`。
- `GET /api/v1/meta/contracts` 与 `GET /api/v1/admin/live-snapshot` 可匿名访问，用于检查线上接口版本和实时硬件快照。
- 登录、任务、交接、文件和报告接口需要 Bearer Token，并按服务端角色返回权限结果。
- 当前公网快照已返回 `TASK-001`、设备与遥测信息，说明该地址不是本机的空数据库。

## 实施步骤

1. 将本地管理端的开发环境 API 基础地址切换为公网后端，并保留该环境文件为 Git 忽略文件。
2. 重启 Vite，分别验证 API 契约和实时硬件快照可从当前机器获取。
3. 用后端分配的测试账号验证正式登录、任务监控、交接、文件和报告等受保护功能；无账号时不创建或猜测公网账号。
4. 记录联调结果和仍需后端/组长提供的账号、权限或 CORS 信息；仅在产生可提交的源代码或文档后提交个人 fork。

## 验收标准

- 浏览器中的管理端请求发送至 `47.103.152.175:8080`，而非 `127.0.0.1:8020`。
- 公网实时快照能显示或被接口检查正确读取 `TASK-001` 的最新硬件数据。
- 使用对应角色测试账号后，受保护接口不会因前端地址、前缀或跨域配置失败；若失败，记录精确接口、状态码和响应信息。
