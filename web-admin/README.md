# PC Web 管理端

生物样本冷链转运交接与可信追溯系统的 Vue 3 管理端。保留 `TASK-001` 的 Mock 演示主链路，同时已接入后端正式登录、我的任务列表/搜索/分页、任务创建编辑、设备台账/绑定历史、实时监控、交接、追溯报告、后端 PDF 下载和真实 GNSS 数据适配。

## 启动

1. 在本目录安装依赖：pnpm install
2. 复制 .env.example 为本机 .env.local，默认可保持 Mock 模式。
3. 运行：pnpm dev
4. 浏览器打开终端显示的本机地址。

Mock 模式下启动后选择演示身份；API 模式下显示账号密码登录。API Token 只保存在当前浏览器会话中，刷新页面时会由 `/api/v1/auth/me` 验证；401 会清会话并回到登录页。不要在页面、截图、日志或仓库中展示 Token。

## 展示增强

- **正式登录与角色菜单**：API 模式调用 `/auth/login`、`/me`、`/permissions`、`/logout`，使用服务端角色驱动菜单；Mock 模式保留四种离线演示身份。
- **我的任务与任务表单**：API 模式使用当前账号的 `/tasks` 列表，支持关键词、状态、分页、创建和编辑；不会把固定 `TASK-001` 误当成所有账号都能访问的任务。
- **设备台账与绑定**：发货方/管理员可从 `/devices` 登记设备、绑定/解绑任务、查看绑定历史。设备关联只走专用绑定接口，任务表单不直接写 `device_id`；页面不接收、显示或保存 `device_secret`。
- **客户端主题**：支持浅色、深色、跟随系统三种显示偏好，并只保存浏览器中的外观选择；不保存账号、业务数据或凭据。
- **单任务实时大屏**：管理员可从“实时大屏”进入 `/screen`，显示当前任务的实时环境、已有遥测历史的 ECharts 温湿度趋势、告警和后端确认的交接节点，并可进入浏览器全屏。它不是多任务全局驾驶舱。
- **加载与故障反馈**：单任务页面统一提供加载、空数据、网络断开/接口失败与重试反馈；恢复连接后仍以轮询和后端返回结果为准。
- **追溯报告 PDF**：追溯页优先下载后端 `/trace-report.pdf`，也保留浏览器打印/保存 PDF 的降级入口；不伪造后端报告。
- **地图轨迹入口**：历史遥测返回真实 `lat/lng/accuracy` 时，页面显示相对轨迹和坐标列表；无坐标时明确空态。底图服务仍需团队确定，不会将参与方名称或 Mock 数据冒充真实位置。

## 切换真实后端

远程联调地址由组长提供后，在不提交的 .env.local 中设置：

    VITE_DATA_SOURCE=api
    VITE_API_BASE_URL=https://组长提供的地址

前端只通过这个变量读取基础地址；不要在页面里填写 127.0.0.1 或远程服务器地址。新页面统一调用 `/api/v1` 接口。API 模式登录后先进入“我的任务”；请使用后端分配的账号和其有权访问的 `WD-...` 任务。

## 验证

    pnpm test
    pnpm build

Mock 数据直接导入仓库的 docs/api/mock/ 文件，仅用于查询界面的独立开发；发出、签收和拒收只会调用真实 API，避免在前端复刻任务状态机。页面状态显示使用后端定义的英文枚举映射为中文。真实模式会同时按 HTTP 状态和业务 code 处理错误。

当前仍未完成的重点是：独立样本主数据、设备密钥/关联事务/RBAC 的后端闭环、按任务交接会话详情、批量操作、全局大屏聚合、真实地图底图与文件二进制上传。告警确认/解决 UI 已接基础接口，但最终权限矩阵仍须后端实现。不要自行增加字段或接口；请按交接清单让对应成员先补齐契约与真实数据。
