# PC Web 管理端

生物样本冷链转运交接与可信追溯系统的 Vue 3 管理端。第一阶段覆盖 TASK-001 的演示主链路：任务总览、实时监控、异常查看、发出/签收/拒收和追溯报告；在不改变后端接口契约的前提下，额外提供演示身份、客户端主题、单任务大屏、ECharts 历史趋势、打印/保存 PDF 与地图轨迹接入入口。

## 启动

1. 在本目录安装依赖：pnpm install
2. 复制 .env.example 为本机 .env.local，默认可保持 Mock 模式。
3. 运行：pnpm dev
4. 浏览器打开终端显示的本机地址。

启动后先选择演示身份。该身份只保存在当前浏览器会话中，用于菜单与展示流程；不会采集账号、密码或 Token，也不代表后端已经启用正式鉴权。

## 展示增强

- **演示身份与角色菜单**：提供 `admin`、`sender`、`carrier`、`receiver` 四种契约内角色；后端仍负责交接操作的最终状态校验。
- **客户端主题**：支持浅色、深色、跟随系统三种显示偏好，并只保存浏览器中的外观选择；不保存账号、业务数据或凭据。
- **单任务实时大屏**：管理员可从“实时大屏”进入 `/screen`，显示当前任务的实时环境、已有遥测历史的 ECharts 温湿度趋势、告警和后端确认的交接节点，并可进入浏览器全屏。它不是多任务全局驾驶舱。
- **加载与故障反馈**：单任务页面统一提供加载、空数据、网络断开/接口失败与重试反馈；恢复连接后仍以轮询和后端返回结果为准。
- **追溯报告 PDF**：在追溯报告页点击“打印 / 保存 PDF”，再在系统打印窗口中选择“另存为 PDF”。报告内容全部来自现有追溯接口，不调用或伪造后端 PDF 接口。
- **地图轨迹入口**：当前接口未提供 GNSS 经纬度，因此页面只展示非地理交接路径，并明确提示需要 `latitude`、`longitude`、`timestamp` 后才能显示真实地图轨迹；不会将参与方名称或 Mock 数据冒充真实位置。

## 切换真实后端

远程联调地址由组长提供后，在不提交的 .env.local 中设置：

    VITE_DATA_SOURCE=api
    VITE_API_BASE_URL=https://组长提供的地址

前端只通过这个变量读取基础地址；不要在页面里填写 127.0.0.1 或远程服务器地址。新页面统一调用 /api/v1 接口。

## 验证

    pnpm test
    pnpm build

Mock 数据直接导入仓库的 docs/api/mock/ 文件，仅用于查询界面的独立开发；发出、签收和拒收只会调用真实 API，避免在前端复刻任务状态机。页面状态显示使用后端定义的英文枚举映射为中文。真实模式会同时按 HTTP 状态和业务 code 处理错误。

正式账号登录、JWT/会话鉴权、GNSS 轨迹、多任务全局统计与后端生成 PDF 需要组长确认新的后端接口契约后再接入；当前实现不会自行增加这些字段或接口。
